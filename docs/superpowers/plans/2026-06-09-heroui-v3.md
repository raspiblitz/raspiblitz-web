# HeroUI v2 → v3 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the app from `@heroui/react` v2.8 to v3.1 (a ground-up rewrite: React Aria Components, compound components, CSS-first theming, no provider), preserving behavior and our brand look.

**Architecture:** Big-bang on branch `chore/upgrade-heroui-v3`. The app will not compile between Phase 0 and the end of Phase 5; `npm run tsc 2>&1 | grep -c error` is the burn-down metric. Full gates (`tsc`/`lint`/`build`/`test`) and devtools-MCP visual QA run at the end. Commit after every task.

**Tech Stack:** React 19, Tailwind CSS v4 (`@tailwindcss/vite`, CSS-first, no JS config), Vite, TypeScript 6, react-hook-form, react-i18next, vitest. Design spec: `docs/superpowers/specs/2026-06-09-heroui-v3-design.md`.

**Verification note (TDD adaptation):** This is a library migration, not feature work — there are no new "failing tests" to write per unit. The per-task check is: the change typechecks the touched file(s) (or reduces total `tsc` errors), and existing unit tests for touched components are updated and pass once the app compiles. New behavior is verified by devtools-MCP visual QA in Phase 6.

## Verified v3 API reference (source of truth)

Extracted from the **installed** `@heroui/react@3.1.0` type declarations (not docs). Everything is exported from the single `"@heroui/react"` barrel.

- **Button** — `variant: primary|secondary|tertiary|outline|ghost|danger|danger-soft`; `size: sm|md|lg`; `isPending` (no `isLoading`); **no** `startContent`/`endContent`/`spinner`/`radius` (icons = children; rounding via Tailwind `rounded-*`); `isIconOnly`, `fullWidth`, `isDisabled` kept. Does **not** auto-render a spinner.
- **TextField/Input/InputGroup/Label/Description/FieldError** — value lives on `TextField`: `value`/`defaultValue`/`onChange?: (value: string) => void`, `name`, `isInvalid`, `isDisabled`, `isReadOnly`, `variant: primary|secondary`. `Input` = bare `<input>`. Adornments: `InputGroup` with `InputGroup.Prefix` / `InputGroup.Input` / `InputGroup.Suffix`. Help/error: `<Description>` / `<FieldError>`.
- **Checkbox** — `Checkbox.Control` / `Checkbox.Indicator` / `Checkbox.Content`; `isSelected`/`defaultSelected`/`onChange?: (isSelected: boolean) => void`; `isIndeterminate` kept; no `color`/`size`.
- **RadioGroup / Radio** — `RadioGroup`: `value`/`defaultValue`/`onChange?: (value: string) => void`, `name` **optional**, `orientation`, `variant`. `Radio`: `value: string` **required**; sub-parts `Radio.Control`/`Radio.Indicator`/`Radio.Content`.
- **Select / ListBox** — `Select.Trigger`/`Select.Value`/`Select.Indicator`/`Select.Popover`; items via `ListBox` + `ListBox.Item` (`id?: Key`, `textValue?: string` — set `textValue` when content isn't plain text). Selection: `value`/`defaultValue`/`onChange` (the v2 `selectedKey`/`selectedKeys`/`onSelectionChange` are deprecated). Single: `value: Key|null`.
- **Tabs** — `Tabs.List`/`Tabs.Tab`/`Tabs.Panel`/`Tabs.Indicator`; `selectedKey`/`defaultSelectedKey`/`onSelectionChange?: (key: Key)=>void`; `Tabs.Tab id` must match `Tabs.Panel id`.
- **Modal** — members: `Modal.Trigger`, `Modal.Backdrop`, `Modal.Container`, `Modal.Dialog`, `Modal.Header`, `Modal.Icon`, `Modal.Heading`, `Modal.Body`, `Modal.Footer`, `Modal.CloseTrigger`. Open state: pass the whole `useOverlayState()` object to `<Modal state={state}>`. `isDismissable` is on **`Modal.Backdrop`** (default `true`). `Modal.Container` has `placement`/`scroll`/`size`.
- **useOverlayState** — returns `{ isOpen, setOpen(b), open(), close(), toggle() }` (NO `.state` field — pass the whole object). Props: `{ isOpen?, defaultOpen?, onOpenChange? }`.
- **Accordion** — `Accordion.Item` (`id?: Key`), `Accordion.Heading`/`Accordion.Trigger`/`Accordion.Panel`/`Accordion.Body`; `expandedKeys`/`defaultExpandedKeys`/`onExpandedChange?: (keys: Set<Key>)=>void`.
- **Card** — `Card.Header`/`Card.Title`/`Card.Description`/**`Card.Content`**/`Card.Footer`. ⚠ **No `Card.Body`** (it's `Card.Content`).
- **Dropdown** — `Dropdown.Trigger`/`Dropdown.Popover`/`Dropdown.Menu`/`Dropdown.Item`/`Dropdown.Section`; `variant` on Root/Popover/Menu; item `variant: danger|default`; selection via RAC `selectedKeys`/`onSelectionChange`, per-item `id`/`onAction`.
- **Spinner** — exists; `color: accent|current|danger|success|warning`, `size: sm|md|lg|xl` (⚠ no arbitrary color names).
- **Link** — exists (`Link` + `Link.Icon`); RAC `href`/`target`/`isDisabled`.
- **Tooltip** — `Tooltip.Trigger`/`Tooltip.Content`/`Tooltip.Arrow`; open state on `Tooltip` root (`delay`/`closeDelay`); `Tooltip.Content` has `showArrow`/`offset`.
- **ProgressBar** (was `Progress`) / **ProgressCircle** (was `CircularProgress`) — `value?: number`, `minValue`/`maxValue` (default 0–100), `isIndeterminate`.
- **cn** — still exported from `@heroui/react` (re-export of `tailwind-variants`' `cn`); returns `string | undefined`.
- **CSS** — `@import "@heroui/styles";` (after `@import "tailwindcss";`). `tailwindcss` must come first.

### Pattern BUTTON-VARIANT — `color` (+ v2 `variant`) → v3 `variant`
Mapping (apply per call site; remove the `color` prop, set `variant`):
| v2 props | v3 `variant` |
|---|---|
| `color="primary"` (default/solid) | `primary` |
| `color="secondary"` | `secondary` |
| `color="danger"` | `danger` |
| `color="danger" variant="flat"` | `danger-soft` |
| `variant="bordered"`/`"faded"` | `outline` |
| `variant="light"`/`"flat"` (non-danger) | `tertiary` |
| `variant="ghost"` | `ghost` |
| no color (default neutral) | leave default (`primary`) or `tertiary` per look |
Also: `isLoading`→`isPending`; `startContent`/`endContent` icon → move inside children; drop `radius` (the wrapper handles rounding via className).

### Pattern TEXTFIELD-RHF — `Input` + react-hook-form → `TextField` via `Controller`
v3 `TextField.onChange` is `(value: string) => void`, incompatible with RHF `register()` (event-based). Use `Controller` (its `field.onChange` accepts the value directly). Canonical transform of a registered Input:
```tsx
// before (v2):
<Input
  type="password"
  label={label}
  classNames={{ inputWrapper: "bg-tertiary ..." }}
  isInvalid={!!errors.passfirst}
  errorMessage={errors.passfirst?.message}
  {...register("passfirst", { required: msg, minLength: { value: 8, message: msg2 } })}
/>

// after (v3):
<Controller
  name="passfirst"
  control={control}
  rules={{ required: msg, minLength: { value: 8, message: msg2 } }}
  render={({ field, fieldState }) => (
    <TextField
      className="w-full"
      isInvalid={fieldState.invalid}
      value={field.value ?? ""}
      onChange={field.onChange}
      onBlur={field.onBlur}
      name={field.name}
    >
      <Label>{label}</Label>
      <Input type="password" className="bg-tertiary" />
      <FieldError>{fieldState.error?.message}</FieldError>
    </TextField>
  )}
/>
```
Notes: swap `useForm`'s `register` for `control` (`const { control, ... } = useForm(...)`). The `bg-tertiary` slot class moves from `classNames.inputWrapper` to the `<Input>`/`InputGroup` `className`. For adornments (e.g. AmountInput's unit toggle), wrap with `InputGroup` and put the button in `InputGroup.Suffix`.

### Pattern CHECKBOX — compound
```tsx
// before: <Checkbox isSelected={x} onValueChange={setX}>Label</Checkbox>
// after:
<Checkbox isSelected={x} onChange={setX}>
  <Checkbox.Control><Checkbox.Indicator /></Checkbox.Control>
  <Checkbox.Content>Label</Checkbox.Content>
</Checkbox>
```

### Pattern RADIOGROUP — compound + `onChange`
```tsx
// before: <RadioGroup value={v} onValueChange={setV} classNames={{...}}>...</RadioGroup>
// after:  <RadioGroup value={v} onChange={setV} className={...}>...</RadioGroup>
```
`onValueChange`→`onChange`; `classNames` slot object → `className` on the group/parts.

### Pattern RADIO — `CustomRadio.tsx` (compound + remove `.remove-radio` hack)
v3 Radio is compound and the indicator is an explicit `Radio.Indicator`, so the v2 `.remove-radio > span { visibility:hidden }` hack is **deleted** — simply omit `Radio.Indicator` to hide it. `classNames.base`→`className` on `Radio` (or `Radio.Root`); `data-[selected=true]:border-primary`→`data-[selected=true]:border-accent` (RAC sets `data-selected`); `bg-blue-900 hover:bg-primary-800`→`bg-blue-900 hover:bg-accent-800`. Drop `radioGroup` prop (not a v3 Radio prop). Use `Radio.Content` for the text block.

### Pattern SELECT — `I18nDropdown.tsx`
```tsx
// after:
<Select aria-label="Select language" value={i18n.language}
  onChange={(key) => { if (key && key !== i18n.language) { i18n.changeLanguage(String(key)); saveSettings({ lang: String(key) }); } }}>
  <Select.Trigger className="bg-tertiary"><Select.Value /><Select.Indicator /></Select.Trigger>
  <Select.Popover>
    <ListBox>
      {langs.map((lang) => (
        <ListBox.Item key={lang} id={lang} textValue={t(`language.${lang}`)}>
          {t(`language.${lang}`)}
        </ListBox.Item>
      ))}
    </ListBox>
  </Select.Popover>
</Select>
```
`defaultSelectedKeys={[lng]}`→`value={lng}`; `onChange` now receives the key, not a DOM event (rewrite `selectLangHandler`); `classNames.trigger`→`className` on `Select.Trigger`.

### Pattern TABS — compound + `id`
```tsx
// before: <Tabs selectedKey={k} onSelectionChange={setK}><Tab key="a" title="A">...</Tab></Tabs>
// after:
<Tabs selectedKey={k} onSelectionChange={setK}>
  <Tabs.List><Tabs.Tab id="a">A</Tabs.Tab></Tabs.List>
  <Tabs.Panel id="a">...</Tabs.Panel>
</Tabs>
```
`key`→`id`; `title` prop → `Tabs.Tab` children; panel content moves from inside `Tab` to a sibling `Tabs.Panel` with matching `id`.

### Pattern TOOLTIP — compound
```tsx
// before: <Tooltip content="Copy"><button/></Tooltip>
// after:
<Tooltip>
  <Tooltip.Trigger><button/></Tooltip.Trigger>
  <Tooltip.Content>Copy</Tooltip.Content>
</Tooltip>
```

### Pattern ACCORDION — `id` + compound (`ChannelList.tsx`)
```tsx
<Accordion>
  {items.map((c) => (
    <Accordion.Item key={c.id} id={c.id}>
      <Accordion.Heading><Accordion.Trigger>{title}</Accordion.Trigger></Accordion.Heading>
      <Accordion.Panel>{body}</Accordion.Panel>
    </Accordion.Item>
  ))}
</Accordion>
```

### Pattern MODAL — `ConfirmModal.tsx` (compound + `useOverlayState`)
New disclosure shape from `useModalManager`/`useOverlayState` is `{ isOpen, open, close, toggle, setOpen }`. ConfirmModal:
```tsx
import { Modal, useOverlayState } from "@heroui/react";
type OverlayState = ReturnType<typeof useOverlayState>;

// Props.disclosure: OverlayState  (was UseDisclosureReturn)
// body branch:
return (
  <Modal state={disclosure}>
    <Modal.Backdrop>
      <Modal.Container>
        <Modal.Dialog>
          {headline && <Modal.Header><Modal.Heading>{headline}</Modal.Heading></Modal.Header>}
          <Modal.Body>{children || props.body}</Modal.Body>
          <Modal.Footer>
            <Button onPress={() => disclosure.close()} isDisabled={isLoading}>{cancelText || t("settings.cancel")}</Button>
            <Button variant="primary" onPress={onConfirm} isDisabled={isLoading || !onConfirm} isPending={isLoading}>
              {confirmText || t("settings.confirm")}
            </Button>
          </Modal.Footer>
        </Modal.Dialog>
      </Modal.Container>
    </Modal.Backdrop>
  </Modal>
);
```
The `ConfirmModal.Header/Body/Footer` static aliases now point at `Modal.Header`/`Modal.Body`/`Modal.Footer`. For non-dismissable modals (SyncScreen) set `isDismissable={false}` on `Modal.Backdrop`.

### Pattern DISCLOSURE — `useDisclosure`/`useModalManager` consumers
`useDisclosure()` (v2: `{isOpen, onOpen, onClose, onOpenChange}`) → `useOverlayState()` (v3: `{isOpen, open, close, toggle, setOpen}`). At call sites: `.onOpen()`→`.open()`, `.onClose()`→`.close()`, `onOpenChange`→`setOpen`. `use-modalmanager.ts`:
```ts
import { useOverlayState } from "@heroui/react";
// const disclosure = useOverlayState();
// openModal: disclosure.open();  closeModal: disclosure.close();
```
